Foundry Apps' strategic direction on design (April 2026): build a **reusable AI-driven design workflow** that produces professional, non-generic websites/apps automatically. Each product is a POC for the workflow — optimize for replicable process, test on the current product.

**Why:** David said explicitly: "Orbit is a proof of concept. We should feel free to test out workflows and figure out what works best. If that means we have to run a couple of proof of concepts and run the audit against them, let's do that. I want us to be able to avoid [AI-generic] automatically."

**How to apply:**
- When working on Orbit design, think "would this approach work on QuizForge too?" — if not, the approach is too bespoke to be useful.
- Fully automatic + non-generic is impossible. Automation amplifies average SaaS patterns. The differentiator is the **one-time taste setup per product** (IA audit, domain research, bespoke component definition). Tools execute against that.
- The workflow pattern being developed: (1) domain research + reference study, (2) IA audit + bespoke component language, (3) tool mapping per asset (Midjourney = imagery, Claude Design / Stitch = layout mockups, Claude Code = production), (4) ship + anti-generic critique + iterate.
- When a new AI design tool launches (like Claude Design April 2026), treat it as a candidate for step 3, not a replacement for steps 1–2.
- Document findings from each POC variant so the workflow gets sharper over time. The hand-crafted vs. Claude-Design comparison on the Orbit DashboardHero is the first comparative test.

**Anti-generic criteria** to audit any AI-generated output against:
1. Does it feel domain-specific (e.g. "mission control" for a launch tracker) or like a generic SaaS dashboard?
2. Could a non-designer spot it as AI-generated at a glance?
3. Are the components bespoke to the product's domain, or stock card/list/sidebar?
4. Is there visual hierarchy that reflects what actually matters in the data?

If the output fails any of these, the fix is upstream in the brief, not downstream in the tool.
